# Introduction

O3 est un framework frontend modulaire et extensible pour OpenMRS. Il fournit des composants, des API et des modèles
partagés afin que les équipes puissent créer des interfaces plus rapidement.

## Principes directeurs

O3 est guidé par les principes suivants:

- **Réutiliser plutôt que refaire** - composants, API et modèles partagés pour réduire les efforts dupliqués.
- **Modulaire par conception** - ajouter, retirer ou mettre à jour des modules sans reconstruire le système.
- **Configuration d'abord** - chaque module frontend déclare ses propriétés de configuration.
- **Utilisabilité d'abord** - IU optimisée pour le point de soins et des dossiers cliniques complexes.
- **Stack moderne** - accessible à une communauté mondiale de développeurs.
- **Interopérable** - conçu pour fonctionner avec FHIR et d'autres normes.

## Ce qu'O3 n'est pas

O3 n'est **pas** une nouvelle version de plateforme d'OpenMRS. Il s'exécute sur votre base de données existante et
utilise les mêmes API REST et FHIR que l'IU O2. Bien qu'O3 nécessite des versions minimales de la plateforme et de
certains modules backend, vous n'avez pas besoin de refondre votre backend pour l'utiliser. Pour exécuter O3 sur une
installation existante, suivez cette [recette](/docs/recipes/set-up-o3-instance).

## Stack technologique

### Frontend

- [React](https://react.dev) - bibliothèque JavaScript pour la construction d'interfaces utilisateur
- [TypeScript](https://typescriptlang.org) - sur-ensemble fortement typé de JavaScript qui ajoute des annotations de type optionnelles à JavaScript, ce qui permet une vérification de type statique et un meilleur support d'outils.
- [Système de conception Carbon](https://carbondesignsystem.com) - un système de conception open source d'IBM.
- [Single SPA](https://single-spa.js.org) - routeur JavaScript pour les microservices frontend.
- [Webpack](https://webpack.js.org) - emballeur de modules pour applications JavaScript.

### API backend

- [OpenMRS REST](https://rest.openmrs.org)
- [FHIR](https://om.rs/fhir)
