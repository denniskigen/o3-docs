# Connaissances prérequises

Avant de vous lancer dans la construction avec O3, vous devez au moins maîtriser HTML, CSS et JavaScript. Si vous débutez totalement dans le développement frontend, il est particulièrement important de bien comprendre les bases. Voici une liste des technologies utilisées dans O3 ainsi que des ressources recommandées pour les apprendre.

## HTML & CSS

Une solide compréhension des fondamentaux HTML et CSS est essentielle pour le développement frontend. Voici quelques ressources recommandées :

- [Les bases du HTML sur MDN](https://developer.mozilla.org/fr/docs/Learn/Getting_started_with_the_web/HTML_basics)
- [Les bases du CSS sur MDN](https://developer.mozilla.org/fr/docs/Learn/Getting_started_with_the_web/CSS_basics)
- [CSS Tricks - Un guide complet de Flexbox](https://css-tricks.com/snippets/css/a-guide-to-flexbox/)
- [CSS Grid Garden](https://cssgridgarden.com/#fr) - Un jeu interactif pour apprendre CSS grid
- [Le cours CSS de Josh Comeau](https://www.joshwcomeau.com/css/interactive-guide-to-flexbox/)
- [Apprendre CSS sur web.dev](https://web.dev/learn/css/)


## JavaScript

La connaissance de JavaScript est un prérequis pour tout le reste. Nous recommandons les ressources suivantes :

- [Introduction to JavaScript - Codecademy](https://www.codecademy.com/learn/introduction-to-javascript)

- [The Modern JavaScript Tutorial](https://javascript.info/)
- [JavaScript for impatient programmers](https://exploringjs.com/impatient-js/toc.html)
- [Josh Comeau's Operator lookup](https://www.joshwcomeau.com/operator-lookup/)
- [Visual guide to references in JavaScript](https://daveceddia.com/javascript-references/)

Une bonne compréhension des concepts de base est essentielle pour apprendre à développer des modules frontend de haute qualité dans O3. Nous recommandons également les ressources suivantes :

- [Écrire des composants résilients - Dan Abramov (daté mais toujours très utile)](https://overreacted.io/writing-resilient-components/)
- [Un guide (presque) complet du comportement de rendu de React - Mark Erikson](https://blog.isquaredsoftware.com/2020/05/blogged-answers-a-mostly-complete-guide-to-react-rendering-behavior/)
- [Pourquoi React Re-renders - Josh Comeau](https://www.joshwcomeau.com/react/why-react-re-renders/)
- [Guide du débutant à React - Kent C. Dodds (daté mais toujours très utile)](https://egghead.io/courses/the-beginner-s-guide-to-react)
- [Un guide visuel du rendu de React](https://alexsidorenko.com/blog/react-render-cheat-sheet/)
- [Guide des re-rendus de React - Nadia Makarevich](https://www.developerway.com/posts/react-re-renders-guide)
- [Quand React rend-il votre composant - Zhenghao Hao](https://www.zhenghao.io/posts/react-rerender)

## Outils JavaScript

- [yarn](https://yarnpkg.com/) - de nombreux dépôts OpenMRS utilisent yarn comme gestionnaire de paquets. En général, si le dépôt que vous consultez contient un fichier `yarn.lock` à sa racine, il utilise yarn. Nous utilisons les [workspaces de yarn](https://yarnpkg.com/features/workspaces) pour versionner et publier les packages dans nos monorepos.
- [turbo](https://turbo.build/repo) - nous utilisons `turbo` pour exécuter des scripts pendant le développement local et nos processus d'intégration continue.
- [swc](https://swc.rs/) - nous utilisons SWC pour compiler notre code JavaScript et TypeScript dans un format pris en charge par tous les principaux navigateurs.
- [webpack](https://webpack.js.org) - nous utilisons webpack pour empaqueter notre code en vue de la production. Nous utilisons [webpack 5](https://webpack.js.org/blog/2020-10-10-webpack-5-release/), qui est la dernière version de webpack au moment de la rédaction.

## Git

Il est impératif que vous conserviez votre code dans un système de contrôle de version. OpenMRS utilise [Git](https://git-scm.com/). Vous devez connaître les bases de l'utilisation de Git et de GitHub. Voici d'excellentes ressources d'apprentissage :

- Apprenez avec [Git Branching](https://learngitbranching.js.org/)
- [Git et GitHub pour les débutants ](https://www.youtube.com/watch?v=RGOj5yH7evk)
- [Dangit, Git](https://dangitgit.com/en)
- [Guide OpenMRS pour utiliser Git](https://wiki.openmrs.org/display/docs/Using+Git)
