# Migration vers Rspack et Vitest

<Callout type="info">
  Cette page n'est pas encore traduite en français. En attendant, veuillez consulter la [version anglaise](/en-US/docs/migrate-to-rspack-and-vitest) du guide.
</Callout>

Ce guide explique comment migrer le bundler d'un module frontend de Webpack vers [Rspack](https://rspack.dev), et le runner de tests de Jest vers [Vitest](https://vitest.dev). Dans les dépôts OpenMRS à package unique, ces deux migrations ont généralement été livrées sous forme de PRs adjacentes (les monorepos les ont effectuées séparément).

Si votre module n'est pas encore sur Core v6, commencez par là : voir [Migration vers Core v6](/docs/migrate-to-core-v6). Le guide Core v6 laisse délibérément votre module sur Jest et Webpack ; ce guide prend le relais.

Pour les détails complets (raison de la migration, étapes, configuration, pièges courants, considérations monorepo, PRs de référence), reportez-vous à la [version anglaise](/en-US/docs/migrate-to-rspack-and-vitest).
